# User Personas

---

## Persona 1: 임직원 (요청자)

| 항목 | 내용 |
|---|---|
| **이름** | 김사원 |
| **역할** | 사내 임직원 (IT 문의 요청자) |
| **규모** | 약 5,000명 |
| **특성** | 다양한 사업부 소속, IT 전문 지식 수준 다양, 업무 중 빠른 해결 필요 |
| **목표** | IT 문제를 빠르고 정확하게 해결하여 업무 복귀 |
| **Pain Point** | 답변 대기 시간, 담당자 찾기 어려움, 처리 상태 불투명 |
| **접점** | 웹 화면 문의 입력, 이메일 문의, 히스토리 조회 |

## Persona 2: 1차 처리자

| 항목 | 내용 |
|---|---|
| **이름** | 박상담 |
| **역할** | IT Help Desk 1차 상담 인력 |
| **규모** | 4~5명 |
| **특성** | 각 사업부 매뉴얼 기반 교육 이수, AI가 처리 못한 건을 수동 처리 |
| **목표** | AI Fallback 건을 신속히 처리하거나 적절한 2차 처리자에게 분배 |
| **Pain Point** | 반복 문의 처리 부담, 적절한 2차 담당자 파악 어려움 |
| **접점** | 처리자 화면 (미처리 큐), Public/Private 메시지 작성 |

## Persona 3: 2차 처리자

| 항목 | 내용 |
|---|---|
| **이름** | 이전문 |
| **역할** | 각 사업부 소속 IT 담당자 (본업 + 문의 처리 겸임) |
| **규모** | 여러 부서, 담당자 단위 배정 (복수 가능) |
| **특성** | 본인 업무가 있는 상태에서 문의 처리, 전문 영역 한정 |
| **목표** | 본인 담당 문의를 정확히 처리하고 빠르게 업무 복귀 |
| **Pain Point** | 본인 업무 아닌 건 수신, 처리 시간 부담, 답변 작성 부담 |
| **접점** | 분배 알림 수신 (이메일/화면), Private 메시지 작성, "본인 아님" 처리 |

## Persona 4: 관리자

| 항목 | 내용 |
|---|---|
| **이름** | 최관리 |
| **역할** | IT 운영팀 관리자 |
| **규모** | 소수 (1~3명) |
| **특성** | 전체 시스템 운영 책임, 통계/성과 관리, 담당자/부서 구성 관리 |
| **목표** | 시스템 효율 극대화, KPI 달성 (1차 해결률 70%+, 분배 성공률 90%+) |
| **Pain Point** | 운영 현황 파악 어려움, LLM 비용 관리, KB 품질 유지 |
| **접점** | 관리자 대시보드, 통계, 담당자/부서 CRUD, KB 관리, 피드백 관리 |
