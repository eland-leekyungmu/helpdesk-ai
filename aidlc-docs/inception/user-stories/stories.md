# User Stories

> **형식**: Persona-Based + User Journey Hybrid  
> **세분화**: Feature 수준 (중간 단위)  
> **Acceptance Criteria**: 혼합 (핵심 시나리오 Given-When-Then, 나머지 체크리스트)  
> **우선순위**: P1(필수) / P2(중요) / P3(있으면 좋음)

---

## Epic 1: 문의 접수 및 AI 자동 응답

### US-1.1: 웹 화면 문의 입력 [P1]
**As a** 임직원(김사원)  
**I want to** 웹 화면에서 자연어로 IT 문의를 입력하고 파일/이미지를 첨부할 수 있다  
**So that** 언제든 IT 문제를 접수할 수 있다

**Acceptance Criteria:**
- Given 임직원이 로그인한 상태에서
- When 문의 입력 화면에 자연어 텍스트를 작성하고 제출하면
- Then 문의가 시스템에 접수되고 티켓 번호가 발급된다
- [ ] 파일/이미지 첨부 가능 (다중 파일)
- [ ] 입력 필드 유효성 검증 (빈 문의 방지)
- [ ] 접수 확인 메시지 표시

### US-1.2: AI 자동 답변 수신 [P1]
**As a** 임직원(김사원)  
**I want to** 문의 제출 후 AI가 자동으로 답변을 생성하여 즉시 받을 수 있다  
**So that** 대기 없이 문제를 해결할 수 있다

**Acceptance Criteria:**
- Given 문의가 접수된 상태에서
- When AI가 RAG 기반으로 유사 답변을 검색하고 신뢰도가 임계값 이상이면
- Then 5초 이내에 답변이 생성되어 화면에 표시된다
- [ ] 답변 근거(출처 문서) 표시
- [ ] 텍스트 전용 문의 → 경량 LLM 호출
- [ ] 파일/이미지 포함 문의 → 고성능 LLM 호출
- [ ] 답변 유용성 피드백 버튼 (👍/👎)

### US-1.3: AI 카테고리 추천 [P2]
**As a** 임직원(김사원)  
**I want to** AI가 내 문의의 카테고리를 자동 추천해 준다  
**So that** 별도로 분류를 고민하지 않아도 된다

**Acceptance Criteria:**
- [ ] 문의 입력 시 AI가 카테고리를 추천 표시
- [ ] 임직원은 추천을 수락하거나 무시 가능
- [ ] 카테고리 미지정 시에도 문의 접수 가능

### US-1.4: 답변 불가 시 내부 에스컬레이션 [P1]
**As a** 시스템  
**I want to** AI가 답변을 못 찾으면 자동으로 1차 처리자에게 전달한다  
**So that** 요청자 개입 없이 적절한 처리자가 답변을 준비한다

**Acceptance Criteria:**
- Given AI 신뢰도가 임계값 미만인 경우
- When 시스템이 1차 처리자에게 에스컬레이션하면
- Then 요청자에게는 별도 안내 없이 내부적으로 1차 처리자 큐에 추가된다
- [ ] 요청자에게 "전달되었습니다" 등의 메시지 노출하지 않음
- [ ] 1차 처리자가 답변하면 그때 요청자에게 Public 답변 전달
- [ ] 에스컬레이션 사유(신뢰도 점수 등)는 Private 노트로 기록

## Epic 2: 티켓 히스토리 조회

### US-2.1: 본인 문의 목록 조회 [P1]
**As a** 임직원(김사원)  
**I want to** 사이트에 접속하여 내가 접수한 문의 목록을 볼 수 있다  
**So that** 과거 문의와 현재 진행 상태를 확인할 수 있다

**Acceptance Criteria:**
- [ ] 본인 티켓만 목록 표시 (타인 티켓 접근 불가)
- [ ] 티켓 상태 표시 (접수/진행중/완료)
- [ ] 최신순 정렬
- [ ] 검색/필터 기능

### US-2.2: 티켓 상세 조회 (Public 메시지만) [P1]
**As a** 임직원(김사원)  
**I want to** 특정 티켓의 대화 이력을 시간순으로 볼 수 있다  
**So that** 답변 내용과 진행 상황을 파악할 수 있다

**Acceptance Criteria:**
- Given 임직원이 본인 티켓 상세를 조회할 때
- When 메시지 이력이 표시되면
- Then Public 메시지만 시간순으로 노출된다
- [ ] Private 메시지 절대 비노출
- [ ] 첨부파일 다운로드 가능
- [ ] 티켓 상태 및 담당자 배정 결과 확인

## Epic 3: 에스컬레이션 처리 (1차 처리자)

### US-3.1: 미처리 문의 큐 확인 [P1]
**As a** 1차 처리자(박상담)  
**I want to** AI가 처리하지 못한 문의 목록을 큐 형태로 확인할 수 있다  
**So that** 우선순위에 따라 처리할 수 있다

**Acceptance Criteria:**
- [ ] AI Fallback 건만 큐에 표시
- [ ] 접수 시간순 정렬
- [ ] 문의 내용 미리보기
- [ ] AI 판단 근거(신뢰도 점수) 표시 (Private 정보)

### US-3.2: 직접 답변 작성 (Public) [P1]
**As a** 1차 처리자(박상담)  
**I want to** 요청자에게 직접 Public 답변을 작성할 수 있다  
**So that** 간단한 문의를 즉시 해결할 수 있다

**Acceptance Criteria:**
- Given 1차 처리자가 문의를 확인한 상태에서
- When Public 메시지로 답변을 작성하면
- Then 요청자에게 답변이 전달되고 티켓 상태가 업데이트된다
- [ ] Public/Private 토글 선택 가능
- [ ] 답변 템플릿/매뉴얼 참조 기능

### US-3.3: 2차 처리자에게 수동 분배 [P1]
**As a** 1차 처리자(박상담)  
**I want to** 문의를 적절한 2차 처리자에게 수동으로 분배하면서 메시지를 함께 전달할 수 있다  
**So that** 2차 처리자가 맥락을 파악하고 처리할 수 있다

**Acceptance Criteria:**
- [ ] 담당자 목록에서 선택하여 분배
- [ ] 분배 시 메시지(이관 코멘트) 입력 가능 (Private)
- [ ] 분배 후 티켓 상태 변경 (진행중)
- [ ] 분배 이력 로그 저장

---

## Epic 4: 2차 분배 수신 및 처리

### US-4.1: 분배된 문의 수신 [P1]
**As a** 2차 처리자(이전문)  
**I want to** 나에게 분배된 문의를 화면 또는 이메일로 수신할 수 있다  
**So that** 처리해야 할 건을 인지할 수 있다

**Acceptance Criteria:**
- [ ] 화면에서 본인 분배 건 목록 확인
- [ ] 이메일로 분배 알림 수신
- [ ] 문의 내용 + 이관 코멘트(Private) 확인 가능

### US-4.2: Private 답변 작성 [P1]
**As a** 2차 처리자(이전문)  
**I want to** 시스템 화면에서 직접 작성하거나 이메일로 회신하여 Private 메시지를 남길 수 있다  
**So that** AI가 문맥만 수정하여 요청자에게 Public으로 전달한다

**Acceptance Criteria:**
- Given 2차 처리자가 답변을 작성할 때
- When 시스템 화면에서 직접 작성하거나, 분배 알림 이메일에 회신하면
- Then 모두 Private 메시지로 저장되고, AI가 문맥만 수정(내용 변경 금지)하여 Public으로 요청자에게 전달된다
- [ ] 시스템 화면 직접 작성: Private으로 저장
- [ ] 이메일 회신: 인바운드 파싱 후 Private으로 저장
- [ ] Public 메시지 작성 불가 (두 채널 모두)
- [ ] 첨부파일 포함 가능 (두 채널 모두)

### US-4.3: "본인 업무 아님" 처리 [P1]
**As a** 2차 처리자(이전문)  
**I want to** 본인 담당이 아닌 문의를 "본인 아님"으로 처리할 수 있다  
**So that** 올바른 담당자에게 재분배된다

**Acceptance Criteria:**
- [ ] "본인 아님" 버튼 클릭으로 처리
- [ ] 담당자 추천 입력 (선택사항)
- [ ] AI가 재분배 또는 1차 처리자에게 반환
- [ ] 한 번 "본인 아님" 처리된 건은 동일 담당자에게 재분배 금지

## Epic 5: 이메일 티켓 처리

### US-5.1: 이메일 문의 자동 인입 [P1]
**As a** 임직원(김사원)  
**I want to** 이메일로 문의를 보내면 웹 화면 접수와 동일하게 처리된다  
**So that** 별도 사이트 접속 없이 문의할 수 있다

**Acceptance Criteria:**
- Given 임직원이 지정된 Help Desk 이메일로 문의를 보내면
- When 시스템이 이메일을 수신하면
- Then 웹 화면 접수와 동일한 티켓이 생성되고 AI 처리 흐름을 탄다
- [ ] 첨부파일 파싱
- [ ] 발신자 이메일로 사용자 식별

### US-5.2: 이메일 스레드 기반 티켓 식별 [P1]
**As a** 시스템  
**I want to** 이메일 헤더(Message-ID, In-Reply-To, References)로 기존 티켓을 식별한다  
**So that** 동일 문의의 후속 대화가 하나의 티켓으로 관리된다

**Acceptance Criteria:**
- [ ] 제목/헤더 기반 티켓 ID 매칭
- [ ] 신규 이메일 → 신규 티켓 생성
- [ ] 기존 티켓 회신 → 해당 티켓에 메시지 추가

### US-5.3: 최종 답변 후 재문의 처리 [P2]
**As a** 시스템  
**I want to** 최종 답변 이메일에 대한 재문의는 AI를 거치지 않고 이전 2차 처리자에게 직접 재분배한다  
**So that** 동일 맥락의 후속 문의가 효율적으로 처리된다

**Acceptance Criteria:**
- Given 티켓이 완료 상태이고 최종 답변이 이메일로 발송된 후
- When 요청자가 해당 이메일에 회신하면
- Then AI 응답 없이 이전 답변한 2차 처리자에게 자동 재분배된다
- [ ] 티켓 상태 재오픈

### US-5.4: 이메일 답변 발송 [P1]
**As a** 시스템  
**I want to** 이메일로 접수된 문의의 답변을 이메일로 회신한다  
**So that** 요청자가 접수한 채널로 답변을 받는다

**Acceptance Criteria:**
- [ ] Public 메시지만 이메일로 발송
- [ ] Private 메시지 발송 절대 차단 (서버 측)
- [ ] 티켓 ID가 이메일 제목/헤더에 포함

## Epic 6: 관리자 대시보드 및 통계

### US-6.1: 티켓 현황 대시보드 [P1]
**As a** 관리자(최관리)  
**I want to** 전체 티켓 현황을 한눈에 볼 수 있는 대시보드가 있다  
**So that** 운영 상태를 실시간으로 파악할 수 있다

**Acceptance Criteria:**
- [ ] 상태별 집계 (접수/진행중/완료)
- [ ] 담당자별 처리 건수
- [ ] 미처리 티켓 강조
- [ ] Public + Private 전체 조회 가능

### US-6.2: KPI 통계 조회 [P1]
**As a** 관리자(최관리)  
**I want to** 1차 자동 응답률, 2차 분배 성공률 등 KPI를 확인할 수 있다  
**So that** 시스템 성과를 측정하고 개선할 수 있다

**Acceptance Criteria:**
- [ ] 1차 AI 자동 응답 해결률 표시
- [ ] 2차 분배 성공률 표시
- [ ] 처리 시간 분포
- [ ] 기간별 필터 (일/주/월)

### US-6.3: LLM 비용 통계 [P1]
**As a** 관리자(최관리)  
**I want to** LLM 호출 비용을 모델별/기간별로 확인할 수 있다  
**So that** 비용을 관리하고 최적화할 수 있다

**Acceptance Criteria:**
- [ ] 모델별 호출 건수
- [ ] 모델별 토큰 사용량
- [ ] 기간별 비용 집계
- [ ] 비용 추이 차트

### US-6.4: 담당자/부서 관리 [P1]
**As a** 관리자(최관리)  
**I want to** 2차 처리자 담당자와 부서를 추가/수정/삭제할 수 있다  
**So that** 조직 변경에 유연하게 대응할 수 있다

**Acceptance Criteria:**
- [ ] 담당자 CRUD (이름, 이메일, 담당 영역)
- [ ] 부서 CRUD
- [ ] 담당자-부서 매핑
- [ ] 여러 담당자를 동일 영역에 배정 가능

## Epic 7: 인증 및 권한 관리

### US-7.1: 사용자 로그인 [P1]
**As a** 모든 사용자  
**I want to** 이메일과 비밀번호로 로그인할 수 있다  
**So that** 시스템에 안전하게 접근할 수 있다

**Acceptance Criteria:**
- Given 등록된 사용자가 올바른 자격 증명을 입력하면
- When 로그인 버튼을 클릭하면
- Then 역할에 맞는 화면으로 이동한다
- [ ] 비밀번호 bcrypt 해싱
- [ ] 로그인 시도 횟수 제한
- [ ] JWT 토큰 기반 세션

### US-7.2: 역할 기반 접근 제어 [P1]
**As a** 시스템  
**I want to** 사용자 역할(임직원/1차/2차/관리자)에 따라 접근 가능한 기능을 제한한다  
**So that** 권한 없는 접근을 방지한다

**Acceptance Criteria:**
- [ ] 임직원: 문의 입력, 본인 히스토리 조회만 가능
- [ ] 1차 처리자: 미처리 큐, Public/Private 작성, 분배
- [ ] 2차 처리자: 본인 분배 건, Private 작성만
- [ ] 관리자: 전체 기능 접근

## Epic 8: 피드백 루프 및 학습

### US-8.1: 답변 피드백 수집 [P1]
**As a** 임직원(김사원)  
**I want to** AI 답변에 대해 유용성 피드백(👍/👎)을 남길 수 있다  
**So that** AI 답변 품질이 지속적으로 개선된다

**Acceptance Criteria:**
- [ ] 답변마다 👍/👎 버튼 표시
- [ ] 피드백 데이터 저장
- [ ] 중복 피드백 방지 (1회만)

### US-8.2: 학습 데이터 자동 누적 [P2]
**As a** 시스템  
**I want to** 완료된 티켓의 문의-답변 쌍을 학습 데이터로 자동 누적한다  
**So that** RAG 정확도가 점진적으로 향상된다

**Acceptance Criteria:**
- [ ] 티켓 완료 시 문의-답변 데이터 자동 저장
- [ ] 처리자 수정 답변도 학습 데이터 반영
- [ ] 오답 레이블링 기능

### US-8.3: KB 재색인 관리 [P2]
**As a** 관리자(최관리)  
**I want to** Knowledge Base 재색인을 트리거할 수 있다  
**So that** 새로운 학습 데이터가 RAG에 반영된다

**Acceptance Criteria:**
- [ ] 수동 재색인 트리거 버튼
- [ ] 재색인 상태 표시 (진행중/완료)
- [ ] 마지막 색인 시간 표시

## Epic 9: 합성 데이터 생성

### US-9.1: 실 데이터 기반 합성 데이터 생성 [P1]
**As a** 시스템/데이터 엔지니어  
**I want to** 기존 10만 건 실 데이터를 기반으로 합성 질문-답변 데이터를 생성한다  
**So that** RAG Knowledge Base에 충분한 학습 데이터를 확보한다

**Acceptance Criteria:**
- Given 기존 실 데이터(1년치 10만 건)가 입력으로 제공되면
- When 합성 데이터 생성 파이프라인을 실행하면
- Then 동일 유형·패턴의 새로운 질문-답변 쌍이 생성된다
- [ ] 실 데이터 그대로 복제 금지 (paraphrase, 변형 필수)
- [ ] 시나리오 확장 (기존 케이스의 변형)
- [ ] 상상 기반 신규 케이스 생성 가능
- [ ] LLM 활용 데이터 생성

### US-9.2: 합성 데이터 품질 검증 [P1]
**As a** 시스템/데이터 엔지니어  
**I want to** 생성된 합성 데이터의 품질을 검증한다  
**So that** 저품질 데이터가 KB에 유입되지 않는다

**Acceptance Criteria:**
- [ ] 중복 제거
- [ ] 형식 검증 (질문-답변 구조)
- [ ] 카테고리 분류 자동 태깅
- [ ] 품질 점수 기반 필터링

### US-9.3: 합성 데이터 KB 적재 [P1]
**As a** 시스템/데이터 엔지니어  
**I want to** 검증된 합성 데이터를 Bedrock Knowledge Base에 적재한다  
**So that** RAG 검색에 활용할 수 있다

**Acceptance Criteria:**
- [ ] 검증 통과 데이터만 KB에 적재
- [ ] 적재 결과 리포트 (건수, 카테고리별 분포)
- [ ] 적재 후 KB 재색인 자동 트리거
