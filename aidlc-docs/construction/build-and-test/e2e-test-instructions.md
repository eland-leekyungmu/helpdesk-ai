# E2E (End-to-End) Test Instructions

## 목적
실제 사용자 관점에서 전체 워크플로우를 검증합니다.

---

## 테스트 환경 준비
1. 로컬 서버 실행: `npm run dev` → http://localhost:3000
2. DB에 시드 데이터 존재 (테스트 계정)
3. AWS Bedrock / S3 접근 가능

---

## E2E 시나리오 목록

### 🔐 SC-01: 인증 플로우

| # | 단계 | 기대 결과 |
|---|------|-----------|
| 1 | 로그인 페이지 접속 | 이메일/비밀번호 입력 폼 표시 |

| 4 | 잠금 해제 후 재로그인 | 정상 로그인 |

## 테스트 실행 방법

현재 E2E 자동화 프레임워크는 미구축 상태입니다.
위 시나리오를 **수동으로** 브라우저에서 검증하거나,
향후 Playwright/Cypress 도입 시 자동화할 수 있습니다.

### 수동 테스트 체크리스트
```
[ ] SC-01: 인증 플로우
[ ] SC-02: 직원 티켓 생성 → AI 자동 응답
[ ] SC-03: 첨부파일 포함 티켓
[ ] SC-04: 1차 처리자 수동 분배
[ ] SC-05: 2차 처리자 응답 (Private → Public)
[ ] SC-06: 분배 거절 + 재분배
[ ] SC-07: 피드백 + 티켓 종료
[ ] SC-08: 관리자 대시보드
[ ] SC-09: 관리자 사용자/조직 관리
[ ] SC-10: KB 재색인
```
