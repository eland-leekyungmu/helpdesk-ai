## Question 1
문의 접수 후 AI 답변 대기 중 로딩 UX를 선택해주세요.
현재는 버튼에 스피너만 표시됩니다. AI 처리가 수 초 걸릴 수 있어 사용자 이탈 방지가 중요합니다.

A) 단계별 진행 표시 — "AI가 답변을 생성하고 있습니다..." 텍스트 + 프로그레스 바
B) 스켈레톤 UI — 답변 영역에 로딩 플레이스홀더 표시
C) 현재 방식 유지 — 버튼 스피너만
D) Other (직접 기술해주세요)

[Answer]: 
A, 자동 응답인 경우에만 AI가 답변을 생성하고 있다고 하고 다른 경우는 다른 메세지로 해줘

## Question 2
티켓 목록의 빈 상태(Empty State) 처리 방식을 선택해주세요.
현재는 "접수된 문의가 없습니다." 텍스트만 표시됩니다.

A) 일러스트 + 안내 문구 + 액션 버튼 — "아직 문의가 없어요. 첫 문의를 접수해보세요!" + [문의하기] 버튼
B) 아이콘 + 안내 문구만 — 심플하게 아이콘과 텍스트
C) 현재 방식 유지 — 텍스트만
D) Other (직접 기술해주세요)

[Answer]: A

## Question 3

처리자는 어떤 티켓을 먼저 처리할지 빠르게 판단해야 합니다.

A) 색상 좌측 바 + 배지 — 우선순위별 컬러 바(빨강/노랑/초록)와 배지 동시 표시
B) 배지만 — 현재와 동일하게 배지로만 표시
C) 대기 시간 강조 — 오래된 티켓일수록 빨간색으로 강조 표시

## Question 5

B) 인라인 메시지 — 해당 폼/버튼 근처에 성공/실패 메시지 표시
C) 모달 팝업 — 확인 버튼이 있는 다이얼로그

B) 수동 새로고침 버튼 — 우측 상단에 새로고침 버튼 제공

## Question 7

B) 전문적이고 깔끔한 톤 — 다른 역할과 동일한 비즈니스 스타일
